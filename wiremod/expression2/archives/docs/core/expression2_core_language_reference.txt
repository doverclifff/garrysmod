Expression 2 Core Language Reference - adapted from the Expression 2 GitHub wiki

This plain-text document is an adaptation of a sub-set of documents
from the official Expression 2 Language Reference into a single
monolithic plain-text document. These documents were selected
because they describe the core features of the language.

It is intended to be used by artificial intelligence technologies, such as
large language models (LLMs). Some modifications have been made from the
original documents in order to adapt them for AI use.

    0 What is Wiremod?
    1 Wiremod is an addon that was created by Erkle, and is now maintained by the Wire Team. This addon allows the user to spawn gates, sensors, inputs and outputs and wire them together into working, computer-like contraptions in Garry's Mod. The gates are connected with wires, which are beams that act as a medium to carry the output from one wired entity to an input on another.

    0 Expression 2: Self-Aware Functions
    1 With entity() you can use Entity-Support to get all the data from the expression-entity. With concmd() you can execute console commands.
    2 Also, the chip has the ability to force itself. Forces aren't dispersed over a certain amount of time, all forces applied to an object within a tick are added up and then applied to the object. Force commands are best used with runOnTick(NumberType) because you won't end up applying more than 1 force per tick and it is easier to do things like defy gravity.
    3 This page lists all of the available functions used by the selfaware extension. To see examples, check [[Expression 2 Examples: Self Aware]].

    0 Expression 2 has events that allow you to manage what parts of your code run when certain things happen, like a player typing in chat and when a player leaves the server.
    1 They're analogous to gmod's [hook](https://wiki.facepunch.com/gmod/hook) system if you are familiar Garry's Mod Lua.
    2 ❔ Why?
    3 If you're here because of your code giving you warnings about certain runOn* functions being deprecated, this is for you.
    4 The [clk (trigger)](https://github.com/wiremod/wire/wiki/E2:-Triggers) system is the old way of handling "events" with runOn* and *clk() functions. It's being replaced due to being one of the most confusing and suboptimal things about the language.
    5 Events are closer to what real world programming languages do, are more efficient, and simpler to understand.
    6 🔣 Syntax
    7 The syntax is pretty simple. They are similar to functions in how you declare them, but missing the return value type, and using event instead of function. *Note, however, that events are not functions. They are a compile time construct. You can't redeclare them or stop them, at least for now. Check a variable inside the event if you do not want code inside of it to run anymore.*
    8 <start codeblock>
    9 event chat(Ply:entity, Said:string, Team:number) {
    10 print(Ply, " just said ", Said)
    11 }
    12 Owner = owner()
    13 event keyPressed(Ply:entity, Key:string, Down:number, Bind:string) {
    14 if (Ply == Owner & !Down) {
    15 print("Owner released key " + Key)
    16 }
    17 }
    18 <end codeblock>
    19 Additionally, if you just begin to type event  and the name of the event, the editor will autocomplete the whole declaration for you!
    20 📚 How do they work?
    21 All events you write in the code are registered at compile time *once*. So you should nest them inside any if statement.
    22 Once they are registered, every time the event fires, **only the code inside of the event will run**. You can think of them as functions you give to lua.
    23 ⏲️ Timers
    24 For simplicity and efficiency, timers work on a different system than events. See [[E2: Timers]].

    0 >  Expression 2 is its own custom language, its syntax will take some time to get used to. While it does have some syntax inspiration from languages like [C](https://en.wikipedia.org/wiki/C_(programming_language)) or [TypeScript](https://www.typescriptlang.org/) you shouldn't expect the syntax to work the same as those languages.
    1 > Also since Expression 2 is its own language, github highlighted should be taken with a grain of salt. We try to swap in other language's syntax highlighting to help visually convey the concepts but your best course of action is to copy any example code and paste it into the ingame editor.

    12 Directives
    13 Directives are what explain what should be changed about the E2 at compile time.
    14 They Follow this format:
    15 <start codeblock>
    16 @<NAME> <ARGS>
    17 Where <ARGS> is usually a whitespace delimited list of variables and their types. Like so:
    18 @persist A:number B:string [C D E]:entity
    19 <end codeblock>
    20 Here is a table for all Expression2 Directives
    21 | Directive | Use-case |
    22 | :---: | --- |
    23 | @name | Set the name of the chip that shows when hovering over it. Note this does not have to be the same as the filename. |
    24 | @input | This is where you will define sources of information your chip will take in. |
    25 | @output | This is the information your chip sends out to be used in other wire components. |
    26 | @persist | Any Variable placed after this directive will stay defined between multiple chip operations |
    27 | @trigger | The trigger directive can selectively enable or disable inputs from triggering executions. |
    28 | @model | Adding the path to a prop model after this directive will cause your chip model to display as that prop. |
    29 | @strict | This makes the E2 throw runtime errors whenever an internal error occurs. Not including this directive will make the E2 fail silently
    30 | @autoupdate | Using @autoupdate will enable auto updating. What this means is that whenever you paste a duplication of an E2 with autoupdate enabled, the E2 will check your files for a new version of that E2 and upload it. |
    31 > [!IMPORTANT]
    32 > Regarding @autoupdate:
    33 >- Only works on saved E2s.
    34 >- Auto Updating is disabled by default and will only work if @autoupdate is specified.
    35 >- If you for some reason need to get an old version of your E2 from a dupe, you will have to temporarily change the name of your E2 so that the autoupdate feature doesn't find the file. Then it'll silently fail and leave you with the old code.
    36 Comments
    37 - adding a # will comment out the rest of that line
    38 - Multiline comments can be started with #[ and ended with ]#

    40 print("Hello World!") # Prints Hello World!

    42 The print function requires a string
    43 ]#

    45 PreProcessing
    46 The preprocessor commands #ifdef, #ifndef, and #else can be used to determine which code to execute based on whether a function exists or not.
    47 <start codeblock>
    48 ifdef propSpawn(svan)
    49 print("You have propcore!")
    50 else
    51 error("This E2 requires propcore!")
    52 endif
    53 <end codeblock>
    54 Library creators can use the #error and #warning commands in combination with #if(n)def commands to generate compile-time errors or warnings. For instance:
    55 <start codeblock>
    56 ifndef propSpawn(svan)
    57 error This chip cannot run without propcore! Enable the propcore extension!
    58 endif
    59 ifndef superCoolFunction()
    60 warning This E2 won't be as cool without superCoolFunction()
    61 endif
    62 <end codeblock>

    71 DataTypes
    72 There are four fundamental types in Expression2, namely:
    73 *  number - represents numerical data and also truthiness, where any non-zero value is considered "truthy" *(see [Scopes & Conditionals](#scopes--conditionals))*.
    74 *  string - represents a simple string enclosed in quotation marks. It can be multiline.
    75 *  array - represents a sequence of items and can't contain other arrays or tables. Similar to a List in other programming languages.
    76 *  table - represents a typed table that can have arbitrary string and numerical keys. Similar to a Dictionary in other programming languages.
    77 * function - represents a user-defined function that you can call
    78 These four basic types are utilized in virtually every Expression2 script. The table below presents all of the types supported by Expression2.
    79 | Shorthand | Datatype      |
    80 | --------- | ------------- |
    81 | N         | Number        |
    82 | S         | String        |
    83 | R         | Array         |
    84 | T         | Table         |
    85 | F         | Function      |
    86 | V         | Vector        |
    87 | E         | Entity        |
    88 | A         | Angle         |
    89 | RD        | RangerData    |
    90 | B         | Bone          |
    91 | M2        | Matrix2       |
    92 | XWL       | Wirelink      |
    93 | C         | ComplexNumber |
    94 | Q         | Quaternion    |
    95 Unlike dynamic programming languages, Expression2 is a statically-typed language, meaning that variables cannot be assigned to different types.
    96 <start codeblock>
    97 COMPILE ERROR!
    98 Variable = 55
    99 Variable = "Fifty Five"
    100 VALID
    101 MyList = array(1, 2, 3, 4)
    102 MyList = array(5, 6, 7, 8)
    103 <end codeblock>
    104 Number literals can be created in binary or hexadecimal by prefixing the number with 0b for binary or 0x for hexadecimal.
    105 <start codeblock>
    106 print(0b10) # prints "2"
    107 print(0x16) # prints "22"
    108 <end codeblock>
    109 Hexadecimal and unicode can be used in string literals using \xXX and \u{XXXXXX} where \u requires between 1 and 6 hexadecimal characters less than 10ffff.
    110 <start codeblock>
    111 print("\x48\x65\x6c\x6c\x6f\x2c\x20\x57\x6f\x72\x6c\x64\x21") # prints "Hello, World!"
    112 print("\u{2615}") # prints "☕"
    113 <end codeblock>

    121 if (1) {

    136 Operators
    137 Expression2 supports the following operators in the order of precedence (from highest to lowest priority):
    138 | Priority | Operator | Syntax | Description |
    139 | --- | --- | --- | --- |
    140 | 1 | Increment | Var++ | Causes Var to increase by 1 |
    141 | 1 | Decrement | Var-- | Causes Var to decrease by 1 |
    142 | 2 | Literals |  | Hardcoded numbers/strings |
    143 | 2 | Trigger | ~Input | Outputs 1 if the following input caused the current execution |
    144 | 2 | Delta | $Var | Outputs the change in Var since the last call of $Var |
    145 | 2 | Connected Wires | ->InOut | is InOut connect to the wire I/O? |
    146 | 3 | Parenthesis | () | can act as a function call or be used in math equations to control order of operations |
    147 | 3 | Function Calls | function(Var) | Calls said function with given Var |
    148 | 4 | Method Calls | Var:method(...) | calls the method() function on the given Var |
    149 | 4 | Indexing | Var[Value,type] | Will attempt to locate the provided Value with the provided type within the provided Var |
    150 | 5 | Unary plus | +Var | Returns the identity (same value) of Var |
    151 | 5 | Unary minus | -Var | Negates the value of Var |
    152 | 5 | Logical NOT | !Var | if something returns true, attaching this operator will make it return false |
    153 | 6 | Exponentiation | A ^ B | Raised A to the power of B |
    154 | 7 | Multiplication | A * B | Multiplies A by B |
    155 | 7 | Division | A / B | Divides A by B |
    156 | 7 | Modulo | A % B | Returns the remainder of division |
    157 | 8 | Addition | A + B | Adds A and B |
    158 | 8 | Subtraction | A - B | Subtracts A and B |
    159 | 9 | Binary Left Shift | A << B | Equivalent to A * (2^B) |
    160 | 9 | Binary Right Shift | A >> B | Equivalent to floor(A/(2^B)) |
    161 | 10 | Greater Than | A > B | if A's value is more than B |
    162 | 10 | Less Than | A < B | if A's value is less than B |
    163 | 10 | Greater Than or Equal to | A >= B | if A's value is more than or equal to B |
    164 | 10 | Less Than or Equal to | A <= B | if A's value is less than or equal to B |
    165 | 11 | Equals | A == B | if A's value equals B's value |
    166 | 11 | NOT Equals | A != B | if A doesn't equal the value of B |
    167 | 12 | Binary XOR | A ^^ B | Returns the binary XOR of A and B. |
    168 | 13 | Binary AND | A && B | Note that this is reversed with the "classical" & operator |
    169 | 14 | Binary OR | A \|\| B | Note that this is reversed with the "classical" \| operator |
    170 | 15 | Logical AND | A & B | Note that this is reversed with the "classical" && operator |
    171 | 16 | Logical OR | A \| B | Note that this is reversed with the "classical" \|\| operator |
    172 | 17 | Ternary Conditional | A ? B : C | Evaluates to B if A is truthy, otherwise C |
    173 | 17 | Binary Conditional | A ?: B | Equivalent to A ? A : B |
    174 | 18 | Assignment | A = B | The value of A now equals the value of B |
    175 It's important to note that some operators work with other types, such as vector arithmetic with +, -, *, and / (per-component), and string concatenation with + (although format or concat should be used for most concatenation operations).
    176 Scopes & Conditionals
    177 Scopes restrict variables to a particular code block (usually enclosed by curly braces { <CODE HERE> }). The highest level of scope outside of any blocks is called the "global" scope, where @persist, @inputs, and @outputs variables are stored, allowing them to be accessed from anywhere in the code.
    178 > [!Important]
    179 > E2 does not use typical booleans, and instead uses *truthiness* (or [*truth value*](https://en.wikipedia.org/wiki/Truth_value#Computing)). In place of booleans, most E2 expressions use 0 to represent *falsiness* and 1 or non-zero to represent *truthiness*.
    180 >
    181 > However, you're not limited to using numbers. Every built-in type in E2 has a truth value test to it, which determines an object's truthiness. For example, the string "" is falsy while "0" is truthy. Empty arrays and tables are falsy while filled ones are truthy.
    182 >
    183 > Generally, you can expect that for any type, it's falsy value is its default, uninitialized value, also called a *zero* value.
    184 if
    185 An if statement executes a block of code only if a condition is met (i.e., the passed variable is non-zero).
    186 <start codeblock>
    187 if (1) {
    188 print("This will run")
    189 }
    190 <end codeblock>
    191 elseif
    192 By using elseif, you can chain conditions together.
    193 <start codeblock>
    194 if (0) {
    195 print("never runs since 0 does not meet condition")
    196 } elseif (2) {
    197 print("This will run since the first if statement failed AND this if statement passes ")
    198 }
    199 <end codeblock>
    200 else
    201 else is used to handle cases where none of the conditions are met.
    202 <start codeblock>
    203 if (0) {
    204 print("This will not run since (0) is not truthy")
    205 } else {
    206 print("This will always run")
    207 }
    208 <end codeblock>
    209 Switch Statements
    210 A switch statement is similar to an if chain and allows for "fallthrough" behavior. If you omit the break statement after each case, it will execute every case underneath it (even if the condition isn't met) until it finds a break or reaches the end of the switch block.
    211 > [!NOTE]
    212 > For performance reasons, you should avoid using switch statements when possible.
    213 <start codeblock>
    214 switch (Var) {
    215 case 2,
    216 print("Var is 2")
    217 break
    218 case 3,
    219 print("Var is 3")
    220 break
    221 case 4,
    222 print("Var is 4") # Will fall through to the default case as well
    223 default,
    224 print("Var is not 2 or 3")
    225 }
    226 <end codeblock>
    227 <details>
    228 <summary>Equivalent code using if statements</summary>
    229 <start codeblock>
    230 if(Var == 2) {
    231 print("Var is 2")
    232 } elseif(Var == 3) {
    233 print("Var is 3")
    234 } else {
    235 if(Var == 4) {
    236 print("Var is 4")
    237 }
    238 print("Var is not 2 or 3")
    239 }
    240 <end codeblock>
    241 </details>
    242 Loops
    243 for
    244 The for loop consists of three numbers: the first represents the current iteration count, the second represents the iteration count at which the loop should end, and the optional third number represents the optional increment *step* value at each iteration. By default, the *step* count is 1.
    245 <start codeblock>
    246 This will call the code 5 times (1 - 5 inclusive). In this case, the third argument is redundant.
    247 for (I = 1, 5, 1) {
    248 ...
    249 }
    250 <end codeblock>
    251 > [!NOTE]
    252 > If you don't need to keep track of the iteration count in a for loop, you can discard the iteration variable to save a bit of performance.
    253 while
    254 A while loop will execute the code within its block as long as the specified condition is true. It's important to note that while loops can be unstable without perf() included in the conditional statement.
    255 <start codeblock>
    256 while (perf(80)){
    257 ...
    258 }
    259 <end codeblock>
    260 do while
    261 A do while loop is similar to a while loop, but it will always execute its block of code at least once. It checks the condition at the end of each iteration to determine whether to continue or exit the loop.
    262 <end codeblock>ts
    263 do {
    264 ...
    265 } while (perf(80))
    266 <end codeblock>
    267 foreach
    268 The foreach loop allows you to iterate through all the values of an array or table.
    269 <end codeblock>csharp
    270 foreach(Key:number, Value:entity = findToArray()) {
    271 ...
    272 }
    273 <end codeblock>
    274 It's recommended to annotate the type of keys when iterating through a table using foreach. By default, foreach on a table will iterate through string keys. To iterate through number keys, annotate with :number.
    275 <end codeblock>csharp
    276 foreach(Key:string, Value:number = MyTable) {}
    277 <end codeblock>
    278 break
    279 The break keyword allows you to exit a loop prematurely.
    280 <start codeblock>
    281 foreach(K, Ply:entity = players()) {
    282 if (Ply:distance(entity()) < 400) {
    283 break
    284 }
    285 Haven't found a player closer than 400 units yet
    286 }
    287 <end codeblock>
    288 continue
    289 The continue keyword allows you to skip a single iteration of a loop.
    290 <end codeblock>ts
    291 for (I = 1, 5) {
    292 if (I % 2 == 0) {
    293 continue  # Skip even numbers (You should use the for loop step parameter instead, though).
    294 }
    295 }
    296 <end codeblock>
    297 User Defined Functions
    298 Functions help to improve code organization and readability by allowing you to break down code into reusable and modular components.
    299 Basic Functions
    300 A function can be defined using the following syntax:
    301 <end codeblock>js
    302 function helloWorld() {
    303 print("Hello world!")
    304 }
    305 helloWorld()
    306 <end codeblock>

    309 > [!NOTE]
    310 > Function parameters are assumed to be numbers by default, unless you specify a different data type using the format <NAME>:<TYPE>.
    311 > This behavior shouldn't be relied on and is deprecated, though.
    312 <end codeblock>js
    313 function hello(Text:string) {
    314 print("Hello ", Text)
    315 }
    316 hello("World!")

    318 Function Returns
    319 You can specify the return type of a function to let you pass values from it to another piece of code.
    320 <end codeblock>ts
    321 function string compliment(Name:string) {
    322 return "You look nice, " + Name + "!"
    323 }
    324 print(compliment(owner():name()))

    327 > You are required to make sure your function returns at the end of all code paths. The editor will error if you do not. If you're not sure where to place the return, put it at the very end of your function.
    328 Variadic Parameters
    329 Variadic parameters allow you to call a function with a varying amount of parameters.
    330 This is essentially just a function that takes an array, but it allows you to pass the arguments to the function directly rather than needing to wrap the arguments in an array()
    331 <start codeblock>
    332 This function calls printColor with a prefix [neat] .
    333 function printMessage(...Args:array) {
    334 	printColor(  array(vec(0, 172, 0), "[neat] ", vec(255)):add(Args) )
    335 }
    336 printMessage("Cool! Here's a number: ", randint(1, 27), ".")
    337 <end codeblock>
    338 You can also define the variadic parameter as a table, which allows you to use arrays and tables in the function call.
    339 <start codeblock>
    340 Sums all arrays together
    341 function number arraySum(...Args:table) {
    342 local Sum = 0
    343 foreach(_:number, R:array = Args) {
    344 Sum += R:sum()
    345 }
    346 return Sum
    347 }
    348 print(arraySum(array(6, 1), array(11))) # Prints "18"
    349 <end codeblock>
    350 User-Defined Methods
    351 (Formerly called meta-functions)
    352 A method is prefixed by a type followed by a colon. This is functionally identical to a normal function, but looks nicer.
    353 <start codeblock>
    354 function entity:hello() {
    355 print("Hello " + This:name())
    356 }
    357 owner():hello() # Prints "Hello " followed by your username
    358 <end codeblock>
    359 Lambda Functions
    360 Lambdas are functions that can be stored in variables, also known as [first-class functions](https://en.wikipedia.org/wiki/First-class_function). For simplicity, you should use [normal functions](#basic-functions) when you don't need the flexibility of lambdas.
    361 Lambdas are quite simple to use. Its syntax is similar to creating a function, and you call a lambda similarly to a normal function.
    362 <start codeblock>
    363 let MyLambda = function(S:string) {
    364 print(S)
    365 }
    366 MyLambda("Hello, World!") # Prints "Hello, World!"
    367 <end codeblock>
    368 Lambdas automatically determine their return type when you define them. To retrieve the value of a lambda return, you'll need to specify the return type in [brackets] after the lambda call.
    369 <start codeblock>
    370 let Runs = 0
    371 let GetRun = function() {
    372 Runs++
    373 return Runs
    374 }
    375 print(GetRun()[number])
    376 print(GetRun()) # ERROR! GetRun is assumed to return void here
    377 <end codeblock>
    378 Lambdas work just like any other type, so you can pass them to functions and return them. You can also define a lambda without storing them and pass them directly (known as a *function literal*).
    379 <start codeblock>
    380 let Wrapper = function(Func:function) {
    381 return function() { # Create and return function literal
    382 print("Wrapped!")
    383 Func()
    384 }
    385 }
    386 let Wrapped = Wrapper(function() { # Create and pass a function literal
    387 print("Hello, World!")
    388 })[function]
    389 Wrapped() # Prints "Wrapped!" and "Hello, World!"
    390 <end codeblock>
    391 *See also: [E2 Guide: Lambdas](/wiremod/wire/wiki/E2-Guide:-Lambdas)*
    392 Calling functions with strings
    393 > [!WARNING]
    394 > String calling is deprecated and will not be supported in the future. Look above to [Lambda Functions](#lambda-functions) to use the replacement.
    395 Using string calls to dynamically call functions should be avoided due to their performance cost and hacky nature. ~~However, this can serve as a substitute for "delegates" or "lambdas" from other languages until they are available.~~
    396 <start codeblock>
    397 let F = "print"
    398 F("Hello world!") # Same as print("Hello world!")
    399 let X = "health"
    400 let OwnerHealth = X(owner())[number] # Same as owner():health()
    401 <end codeblock>
    402 > [!WARNING]
    403 > These are deprecated and have been superceded by lambdas.

    406 include
    407 The #include syntax in E2 is similar to that of C and C++. When an #include statement is encountered, E2 will execute the code in the specified file. It's recommended to put the #include statement inside an if(first()) statement, and note that despite its appearance, it is parsed like a normal statement and is not related to the preprocessor.
    408 <start codeblock>
    409 example_lib.txt
    410 function doStuff() {
    411 print("Hello world!")
    412 }
    413 <end codeblock>
    414 <start codeblock>
    415 if (first()) {
    416 include "example_lib"
    417 ^^ make sure not to include .txt ^^
    418 Now you can use functions and global variables from the "example_lib" file.
    419 doStuff()
    420 }
    421 <end codeblock>
    422 Exception Handling
    423 By default, Expression 2 does not often throw errors; instead, it will silently fail and return a default value on function calls. However, if you enable the @strict directive, these errors will be thrown as proper runtime errors. To handle them, you should use try and catch blocks.
    424 <start codeblock>
    425 try { # < -- If an error occurs in this block, it will exit the block and call the catch {} block, passing the error message string.
    426 error("Unexpected error")
    427 } catch(E:string) { # < -- Can rename E to anything. It just stores the error message.
    428 assert(E == "Unexpected error")
    429 }
    430 <end codeblock>
    431 However, it's important to note that this only covers errors that occur within Expression 2 and not internal Lua errors caused by E2 extensions.

    433 *Main article: [Expression 2 Events](/wiremod/wire/wiki/Expression-2-Events)*

    437 print(Ply, " just said ", Said)
    438 }

    440 event keyPressed(Ply:entity, Key:string, Down:number, Bind:string) {
    441 if (Ply == Owner & !Down) {
    442 print("Owner released key " + Key)
    443 }
    444 }
    445 <end codeblock>
